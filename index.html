Create bot

Open Telegram → search @BotFather → /newbot → save the BOT_TOKEN.

2
Get user's chat_id

Have the user send /start to your bot. Use getUpdates to capture their chat_id, or use a deep-link: https://t.me/YourBot?start=USER_ID.

3
Send reminders
import requests, os
TOKEN = os.environ["TG_BOT_TOKEN"]

def remind(chat_id, bank, amount, due_date, days_left):
    requests.post(
        f"https://api.telegram.org/bot{TOKEN}/sendMessage",
        json={
            "chat_id": chat_id,
            "parse_mode": "HTML",
            "text": (
                f"⏰ <b>Settlement Reminder</b>\n"
                f"Bank: {bank}\n"
                f"Amount: ₴{amount:,.0f}\n"
                f"Due: {due_date} ({days_left}d left)\n"
                f"Settle to avoid interest!"
            )
        }
    )

Step 3 — Daily Cron + Serverless Function
1
Cloudflare Worker (or Vercel Edge Function)

Free tier: 100K requests/day. Store user data in Cloudflare KV or D1 SQLite (free). The worker checks each user's schedule and sends notifications.

// Cloudflare Worker — wrangler.toml
name = "grace-reminder"
main = "src/index.js"
compatibility_date = "2024-01-01"

[[kv_namespaces]]
binding = "USERS"
id = "your-kv-namespace-id"
2
Cron trigger

Option A: cron-job.org (free) — set a daily HTTP GET to your worker URL.
Option B: Cloudflare Cron Triggers (built-in, free) — add to wrangler.toml:

[triggers]
crons = ["0 8 * * *"]  # every day at 08:00 UTC
3
Worker logic
export default {
  async scheduled(event, env) {
    const users = await env.USERS.list();
    const today = new Date();
    
    for (const key of users.keys) {
      const user = JSON.parse(await env.USERS.get(key.name));
      
      for (const card of user.cards) {
        const due = new Date(card.nextSettlement);
        const daysLeft = Math.ceil((due - today) / 86400000);
        
        if (daysLeft <= 3 && daysLeft >= 0) {
          await sendTelegram(user.chatId, card, daysLeft);
          await sendEmail(user.email, user.name, card, daysLeft);
        }
      }
    }
  }
};

Step 4 — Automatic OVDP Auction Data
1
Source: National Bank of Ukraine (NBU)

The NBU publishes OVDP auction results at bank.gov.ua. Use their open data API to fetch the latest yields:

# NBU OVDP auction results API
import requests
from datetime import datetime

def get_ovdp_auctions():
    url = "https://bank.gov.ua/NBUStatService/v1/statdirectory/ovdp"
    params = {
        "period": "daily",
        "date": datetime.now().strftime("%Y%m%d"),
        "json": ""
    }
    r = requests.get(url, params=params)
    return r.json()

# Also: Ministry of Finance auction calendar
# https://mof.gov.ua/en/ozdp — scrape or use RSS feed
# for upcoming auction dates & planned volumes
2
Match bonds to user's cycle length

Filter OVDP by maturity ≤ user's grace period. Rank by effective yield. Include in the daily notification:

def suggest_bonds(grace_days, auctions):
    # Filter bonds with maturity fitting the grace cycle
    matching = [
        a for a in auctions
        if a.get("repayPeriod", 999) <= grace_days - 1
    ]
    # Sort by weighted average yield descending
    matching.sort(
        key=lambda a: a.get("auctionRate", 0),
        reverse=True
    )
    return matching[:3]  # top 3 picks

Auction calendar alerts

Scrape or poll the Ministry of Finance auction calendar weekly. When a new auction is announced with maturity matching a user's cycle, push a Telegram notification:

def check_upcoming_auctions(env):
    # Poll MoF auction calendar (weekly cron)
    calendar = fetch_mof_calendar()
    users = get_all_users(env)
    
    for auction in calendar:
        for user in users:
            for card in user["cards"]:
                if auction["maturity_days"] <= card["grace"] - 1:
                    notify(user, (
                        f"📊 New OVDP auction on {auction['date']}\n"
                        f"Maturity: {auction['maturity_days']}d "
                        f"(fits your {card['bank']} cycle)\n"
                        f"Expected yield: ~{auction['est_rate']}%"
                    ))

5. Write worker logic (see Step 3 above)
6. wrangler deploy
7. Done — daily reminders, email alerts, and OVDP suggestions all running.
